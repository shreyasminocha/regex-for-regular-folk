---
title: Character Escapes
---

Character escapes act as shorthands for some common character classes.

## Digit character — `\d`

The character escape `\d` matches digit characters, from `0` to `9`. It is equivalent to the character class `[0-9]`.

<Example regex={/\d/g}>
    <li>2020</li>
    <li>100/100</li>
    <li>It costs $5.45</li>
    <li>3.14159</li>
</Example>

<Example regex={/\d\d/g}>
    <li>2020</li>
    <li>100/100</li>
    <li>It costs $5.45</li>
    <li>3.14159</li>
</Example>

<Note>
    While <code>59</code> is also a pair of digits, most engines look for{" "}
    <em>non-overlapping</em> matches from left to right by default.
</Note>

`\D` is the negation of `\d` and is equivalent to `[^0-9]`.

<Example regex={/\D/g}>
    <li>2020</li>
    <li>100/100</li>
    <li>It costs $5.45</li>
    <li>3.14159</li>
</Example>

## Word character — `\w`

The escape `\w` matches characters deemed "word characters". These include:

-   lowercase alphabet — `a`–`z`
-   uppercase alphabet — `A`–`Z`
-   digits — `0`–`9`
-   underscore — `_`

It is thus equivalent to the character class `[a-zA-Z0-9_]`.

<Example regex={/\w/g}>
    <li>john_s</li>
    <li>matej29</li>
    <li>Ayesha?!</li>
    <li>4952</li>
    <li>LOUD</li>
    <li>lo-fi</li>
    <li>get out</li>
    <li>21*2 = 42(1)</li>
</Example>

<Example regex={/\W/g}>
    <li>john_s</li>
    <li>Ayesha?!</li>
    <li>4952</li>
    <li>LOUD</li>
    <li>lo-fi</li>
    <li>get out</li>
    <li>;-;</li>
    <li>21*2 = 42(1)</li>
</Example>

## Whitespace character — `\s`

The escape `\s` matches whitespace characters. The exact set of characters matched is dependent on the regex engine, but most include at least:

-   space
-   tab — `\t`
-   carriage return — `\r`
-   new line — `\n`
-   form feed — `\f`

Many also include vertical tabs (`\v`). Unicode-aware engines usually match all characters in the `separator` category.

The technicalities, however, will usually not be important.

<Example regex={/\s/g}>
    <li>word word</li>
    <li>tabs vs spaces</li>
    <li>snake_case.jpg</li>
</Example>

<Example regex={/\S/g}>
    <li>word word</li>
    <li>tabs vs spaces</li>
    <li>snake_case.jpg</li>
</Example>

## Any character — `.`

While not a typical character escape, `.` matches any[^1] character.

<Example regex={/./g}>
    <li>john_s</li>
    <li>Ayesha?!</li>
    <li>4952</li>
    <li>LOUD</li>
    <li>lo-fi</li>
    <li>get out</li>
    <li>;-;</li>
    <li>21*2 = 42(1)</li>
</Example>

[^1]: Except the newline character `\n`. This can be changed using the ["dotAll" flag](/chapters/flags#dot-all-s), if supported by the regex engine in question.
