---
title: Escapes
---

In regex, some characters have special meanings as we will explore across the chapters:

-   `|`
-   `{`, `}`
-   `(`, `)`
-   `[`, `]`
-   `^`, `$`
-   `+`, `*`, `?`
-   `\`
-   `.` — Literal only within character classes.[^1]
-   `-` — _Sometimes_ a special character within character classes.

[^1]: Many special characters that would otherwise have special meanings are treated literally by-default inside character classes.

---

When we wish to match these character _literally_, we need to "escape" them.

This is done by **prefixing the character with a `\`**.

---

<Example regex={/\(paren\)/g}>
    <li>paren</li>
    <li>parents</li>
    <li>(paren)</li>
    <li>a (paren)</li>
</Example>
<Example regex={/(paren)/g}>
    <li>paren</li>
    <li>parents</li>
    <li>(paren)</li>
    <li>a (paren)</li>
</Example>

---

<Example regex={/example\.com/g}>
    <li>example.com</li>
    <li>a.example.com/foo</li>
    <li>example_com</li>
    <li>example@com</li>
    <li>example_com/foo</li>
</Example>
<Example regex={/example.com/g}>
    <li>example.com</li>
    <li>a.example.com/foo</li>
    <li>example_com</li>
    <li>example@com</li>
    <li>example_com/foo</li>
</Example>

---

<Example regex={/A\+/g}>
    <li>A+</li>
    <li>A+B</li>
    <li>5A+</li>
    <li>AAA</li>
</Example>
<Example regex={/A+/g}>
    <li>A+</li>
    <li>A+B</li>
    <li>5A+</li>
    <li>AAA</li>
</Example>

---

<Example regex={/worth \$5/g}>
    <li>worth $5</li>
    <li>worth $54</li>
    <li>not worth $5</li>
</Example>
<Example regex={/worth $5/g}>
    <li>worth $5</li>
    <li>worth $54</li>
    <li>not worth $5</li>
</Example>

## Examples

### JavaScript in-line comments

<Example regex={/\/\/.*/g}>
    <li>console.log(); // comment</li>
    <li>console.log(); // // comment</li>
    <li>console.log();</li>
</Example>

### Asterisk-surrounded substrings

<Example regex={/\*[^\*]*\*/g}>
    <li>here be *italics*</li>
    <li>permitted**</li>
    <li>a*b*c*d</li>
    <li>a*b*c*d*e</li>
    <li>a️bcd</li>
</Example>

The first and last asterisks are literal since they are escaped — `\*`.

The asterisk inside the character class does not necessarily need to be escaped[^1], but I've escaped it anyway for clarity.

The asterisk immediately following the character class indicates [repetition of the character class](/chapters/repetition#zero-or-more), which we'll explore in chapters that follow.
