---
title: Character classes
---

It's possible to match a character from within a set of characters.

<Example regex={/[aeiou]/g}>
    <li>avocado</li>
    <li>brinjal</li>
    <li>onion</li>
    <li>rhythm</li>
</Example>

`/[aeiou]/g` matches all vowels in our input strings.

Here's another example of these in action:

<Example regex={/p[aeiou]t/g}>
    <li>pat</li>
    <li>pet</li>
    <li>pit</li>
    <li>spat</li>
    <li>spot a pet</li>
    <li>bat</li>
</Example>

We match a `p`, followed by one of the vowels, followed by a `t`.

There's an intuitive shortcut for matching a character from within a continuous _range_.

<Example regex={/[a-z]/g}>
    <li>john_s</li>
    <li>matej29</li>
    <li>Ayesha?!</li>
    <li>4952</li>
    <li>LOUD</li>
</Example>
<Warning>
    The regex <code>/[a-z]/g</code> matches only one character. In the example
    above, the strings have{" "}
    <em>several matches each, each one character long</em>. Not one long match.
</Warning>

We can combine ranges and individual characters in our regexes.

<Example regex={/[A-Za-z0-9_-]/g}>
    <li>john_s</li>
    <li>matej29</li>
    <li>Ayesha?!</li>
    <li>4952</li>
    <li>LOUD</li>
</Example>

Our regex `/[A-Za-z0-9_-]/g` matches a single character, which must be (at least) one of the following:

-   from `A-Z`
-   from `a-z`
-   from `0-9`
-   one of `_` and `-`.

We can also "negate" these rules:

<Example regex={/[^aeiou]/g}>
    <li>Umbrella</li>
    <li>cauliflower</li>
    <li>ou</li>
</Example>

The only difference between the first regex of this chapter and `/[^aeiou]/g` is the `^` immediately after the opening bracket. Its purpose is to negate the rules defined within the brackets. We are now saying:

> "match any character that is _not_ any of `a`, `e`, `i`, `o`, and `u`"

## Examples

### Prohibited username characters

<Example regex={/[^a-zA-Z_0-9-]/g}>
    <li>TheLegend27</li>
    <li>WaterGuy12</li>
    <li>Smokie_Bear</li>
    <li>Robert'); DROP TABLE Students;--</li>
</Example>

### Unambiguous characters

<Example regex={/[A-HJ-NP-Za-kmnp-z2-9]/g}>
    <li>foo</li>
    <li>lily</li>
    <li>lI0O1</li>
    <li>unambiguity</li>
</Example>
